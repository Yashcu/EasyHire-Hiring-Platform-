# API Specification Document

## Intern Job Tracking Application

Version: 1.0  
Status: Final  
Base URL: /api/v1  

---

## 1. Introduction

### 1.1 Purpose

This document defines the REST API contract for the Intern Job Tracking Application.

It specifies:

- Endpoints
- HTTP methods
- Request/response formats
- Authentication requirements
- Status codes
- Error responses

All APIs follow REST principles and use JSON format.

---

## 2. Authentication

### 2.1 JWT Authentication

Protected endpoints require:

```text
Authorization: Bearer <JWT_TOKEN>
```

JWT contains:

- user_id
- role
- expiration

---

## 3. Common Response Format

### 3.1 Success Response

```json
{
  "timestamp": "2026-02-21T10:00:00Z",
  "data": { }

```

```json

  "data": {
```

## 4. Authentication Endpoints

### 4.1 Register User

`POST /auth/register`

Request Body:

```json
{
  "email": "user@example.com",
  "password": "StrongPassword123",
  "role": "CANDIDATE",
  "firstName": "John",
  "lastName": "Doe"
}
```

Response:

`201 Created`

```json
{
  "timestamp": "2026-02-21T10:00:00Z",
  "data": {
    "message": "User registered successfully"
  }
}
```

Errors:

- 400 – Validation error
- 409 – Email already exists

---

### 4.2 Login

`POST /auth/login`

Request Body:

```json
{
  "email": "user@example.com",
  "password": "StrongPassword123"
}
```

Response:

`200 OK`

```json
{
  "timestamp": "2026-02-21T10:00:00Z",
  "data": {
    "token": "jwt_token_here"
  }
}
```

Errors:

- 401 – Invalid credentials

`POST /internships`

Authentication: Required (RECRUITER)

Request Body:

```json
{
  "title": "Frontend Intern",
  "description": "React development",
  "location": "Onsite",
  "stipendMin": 8000,
  "stipendMax": 12000,
  "type": "ONSITE"
}
```

Response:

`201 Created`

```json
{
  "timestamp": "2026-02-21T10:00:00Z",
  "data": {
    "message": "Internship created successfully",
    "id": "new_internship_uuid"
  }
}
```

Errors:

- 403 – Unauthorized role
- 400 – Validation error

## 6. Application Endpoints

### 6.1 Apply to Internship

`POST /internships/{id}/apply`

Authentication: Required (CANDIDATE)

Request Body:

```json
{
  "resumeUrl": "https://res.cloudinary.com/.../resume.pdf"
}
```

Response:

`201 Created`

```json
{
  "timestamp": "2026-02-21T10:00:00Z",
  "data": {
    "message": "Application submitted successfully"
  }
}
```

Errors:

- 403 – Not candidate
- 400 – Internship closed
- 409 – Duplicate application

---

### 7.2 Update Application Status

`PATCH /applications/{id}/status`

Authentication: Required (RECRUITER, owner)

Request Body:

```json
{
  "status": "SHORTLISTED"
}
```

Valid Transitions:

- APPLIED → IN_REVIEW, REJECTED
- IN_REVIEW → SHORTLISTED, REJECTED
- SHORTLISTED → OFFERED, REJECTED

Errors:

- 403 – Unauthorized
- 400 – Invalid status transition
- 404 – Application not found

## 9. Pagination Standard

All paginated endpoints accept:

`?page=0&size=10`

Response includes within `data`:

- content
- page
- size
- totalElements
- totalPages

---

## 10. Security Rules Enforcement

- JWT required for protected endpoints.
- Role-based checks enforced.
- Ownership validation enforced in service layer.
- No user ID accepted in request body for authorization.

- Minor additions do not require version bump.
