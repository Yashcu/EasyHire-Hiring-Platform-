# Threat Model

## Intern Job Tracking Application

Version: 1.0  
Status: Final  

---

## 1. Introduction

### 1.1 Purpose

This document identifies potential security threats to the Intern Job Tracking Application and defines mitigation strategies.

The objective is to:

- Identify realistic attack vectors
- Evaluate risk severity
- Document existing safeguards
- Highlight residual risks

This is a lightweight threat model appropriate for a small-scale web application.

---

## 2. System Overview (Security Context)

System Components:

- React Frontend (Browser Client)
- Spring Boot Backend API
- PostgreSQL Database
- Cloudinary (Resume Storage)
- Hosting Environment (Render / Vercel)

Trust Boundaries:

1. Public Internet → Backend API
2. Backend → Database
3. Frontend → Cloudinary
4. Backend → External Config (Secrets)

---

## 3. Assets to Protect

| Asset | Sensitivity Level | Reason |
| ----- | ----------------- | ------ |
| User Passwords | High | Account takeover risk |
| JWT Tokens | High | Authentication bypass risk |
| Application Data | Medium | Privacy concerns |
| Resume URLs | Medium | Personal information |
| Database Integrity | High | System reliability |
| JWT Secret | Critical | Token forgery risk |

---

## 4. Threat Categories (STRIDE-Based)

---

### 4.1 Spoofing Identity

#### Threat 1: Fake Login Attempt

An attacker attempts to log in using guessed credentials.

Impact:

- Account compromise.

Mitigation:

- BCrypt password hashing.
- No plaintext storage.
- Proper authentication checks.

Residual Risk:

- Brute-force attempts are primarily mitigated by IP-based rate limiting on auth endpoints.

---

#### Threat 2: JWT Forgery

An attacker attempts to create a fake JWT.

Impact:

- Unauthorized access to protected endpoints.

Mitigation:

- HMAC signature verification.
- Secret stored securely in environment variables.
- Token expiration enforced.

Residual Risk:

- Secret leakage compromises system.

---

### 4.2 Tampering

#### Threat 3: Duplicate Application Manipulation

User attempts to apply multiple times concurrently.

Impact:

- Data inconsistency.

Mitigation:

- `UNIQUE(candidate_id, internship_id)` constraint.
- Application-level duplicate check.

Residual Risk:

- None significant (DB-level enforcement).

---

#### Threat 4: Resume URL Tampering

User submits malicious external URL instead of Cloudinary link.

Impact:

- Malicious file references.
- Potential phishing vector.

Mitigation:

- Resume URL validation.
- Domain verification (Cloudinary only).

Residual Risk:

- If validation weak, risk increases.

---

### 4.3 Repudiation

#### Threat 5: Recruiter Denies Status Change

Recruiter claims they did not update application status.

Impact:

- Accountability dispute.

Mitigation:

- `application_status_history` table.
- `changed_by` stored with timestamp.

Residual Risk:

- None significant.

---

### 4.4 Information Disclosure

#### Threat 6: IDOR (Insecure Direct Object Reference)

Candidate attempts to access another candidate’s application.

Impact:

- Privacy breach.

Mitigation:

- Ownership validation in service layer.
- Candidate ID derived from JWT, not request parameter.

Residual Risk:

- High if ownership validation is incorrectly implemented.

---

#### Threat 7: Exposed Stack Trace

Internal exception exposed to user.

Impact:

- System internals revealed.

Mitigation:

- Global exception handler.
- No stack traces in production.

Residual Risk:

- Low.

---

#### Threat 8: XSS Leading to JWT Theft

Attacker exploits a cross-site scripting (XSS) vulnerability to steal the JWT from `localStorage`.

Impact:

- Account compromise.

Mitigation:

- React automatically escapes rendered content, preventing standard XSS.
- Strict input validation on the backend.

Residual Risk:

- Acceptable for MVP. For production-grade security, HTTP-only cookies should be used instead of `localStorage`.

---

### 4.5 Denial of Service (DoS)

#### Threat 9: API Flooding

Attacker floods apply endpoint.

Impact:

- Service slowdown or crash.

Mitigation:

- Database constraints prevent duplicate abuse.
- Free-tier hosting limits capacity.

Residual Risk:

- Advanced distributed DoS (DDoS) may surpass simple IP rate limits.
- Acceptable for small-scale use.

---

### 4.6 Elevation of Privilege

#### Threat 10: Role Escalation

Candidate attempts to access recruiter-only endpoint.

Impact:

- Unauthorized modification of data.

Mitigation:

- Spring Security role checks.
- Controller-level authorization.
- Service-level ownership checks.

Residual Risk:

- Low if properly configured.

## 6. Attack Surface Analysis

Public Endpoints:

- Login
- Register
- Internship listing
- Apply endpoint

Protected Endpoints:

- All application and recruiter endpoints

Most critical endpoints:

- Apply to internship
- Update application status

---

## 7. External Dependency Risks

### 7.1 Cloudinary

Risk:

- External service compromise.
- Resume link abuse.

Mitigation:

- Validate resume URLs.
- Store only URL, not file.

---

### 7.2 Hosting Provider

Risk:

- Infrastructure downtime.
- Environment variable exposure.

Mitigation:

- Secure hosting configuration.
- No hardcoded secrets.

---

## 8. Secure Coding Controls

- No raw SQL.
- DTO validation enforced.
- UUID validation.
- Role checks at controller.
- Ownership checks at service.
- Database constraints enforce integrity.

---

## 9. Residual Risks

Accepted Risks for MVP:

- No multi-factor authentication.
- No intrusion detection.
- No anomaly detection.
- No enterprise compliance certification.

These are acceptable given:

- Small user base.
- Non-financial data.
- Educational project scope.

---

## 10. Future Security Enhancements

- Rate limiting via Redis.
- Refresh token rotation.
- MFA for recruiters.
- JWT key rotation.
- Audit logging improvements.
- Security headers (CSP, HSTS).

---

## 11. Conclusion

This threat model identifies:

- Primary attack vectors
- Asset sensitivity
- Mitigation strategies
- Residual risks

The system adequately protects against:

- IDOR
- SQL injection
- Duplicate manipulation
- Unauthorized role access
- Credential exposure

Remaining risks are acceptable within the defined scale and scope of the Intern Job Tracking Application.
